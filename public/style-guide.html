Body paragraph Default voice. Serif (Source Serif 4), 19px, line-height 1.6, ink. Most of your writing should be this.

This is a body paragraph. Most of your prose should look exactly like this — serif, 19px, line-height 1.6. If a sentence doesn't need to be set apart, it shouldn't be.

Inline emphasis Strong (500 weight), em (italic), code (mono on surface), InlineLink (tint-colored with underline), ExternalLink (also tint, possibly with an icon).

Inline emphasis: strong (bold) for stress, italic (em) for terms or named things. Link to another article: InlineLink. Link outbound: ExternalLink. Literal token: code. File path: src/components/Lede.tsx.

H2 — section heading Serif, 28–36px, plain. Format: §N · Title. Used for every major section of an article.

§1 · Section heading example

Each major section of an article opens with one of these. Plain serif, no emoji (the thinking-outside-the-box exception aside).

H3 — sub-heading Serif 500, 26px. For numbered claims or sub-points inside a section. Replaced the older <MicroHeading> component.

1. A numbered claim or sub-point

H3 is for enumerable points within a section. The leading number is just a writing convention; visually it's a plain h3.

Aside Italic indent, 17px, ink-3. For offhand commentary, parenthetical asides, "lol I once said…". No card, no border. Earns its set-apart voice through italic + softer color.
This is an Aside. It feels like a side-thought, not a structured callout. If your set-apart block needs a label, it's probably a Sidebar.
blockquote Italic with a left-border in the article's tint. The single voice for set-apart speech — quoted text (a prompt, transcript, named source) OR your own thesis moments. PullQuote has been retired; blockquote covers both jobs now.
This is a blockquote. The left border + italic says "this is something worth landing on." Use it for quoted text or your own punchy thesis line.
blockquote with attribution Add the source as a line inside the blockquote at the end.
For attribution, just add a line inside the blockquote at the end.

Sidebar Neutral surface card with a mono uppercase kicker label. For structured content where the label is doing real editorial work (a definition, a procedure, a self-contained aside that earns its own title).
When to use a Sidebar
  • The block contains structured content (a list, a sub-flow, a diagram)
  • The block deserves its own title — the kicker label is editorial, not "Example"
  • You'd want the reader to be able to navigate to it as a unit
Unordered list For short itemized lists where order doesn't matter. Plain bullets, indented under the body column.
  • For short itemized lists where order doesn't matter
  • One bullet per item, indented under the body column
  • Avoid nesting more than one level deep
Ordered list For procedures or sequenced steps. Same rhythm as ul, just numbered. Use when the count itself is meaningful.
  1. For procedures or anything where order matters
  2. Same visual rhythm as the unordered list, just numbered
  3. Use this when the count itself is meaningful
Raw <table> For 2–3 column comparison data. Bordered card, rounded corners, surface header row, hairline separators. First column reads at full ink, others at ink-2.
